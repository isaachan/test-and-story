Description

**1. Add a section to Page 1 titled 'Policy Holder details' (as per wireframe)

2. Add a question labeled 'What is the date of birth of the oldest policy holder?'

3. This is a date input field, in the following format: DD/MM/YYYY
(The user can only enter Numeric values. Alpha characters will not be recognised)

4. There is no default value for this field - 'DD/MM/YYYY' should display in a light colour as a guide, but should be removed when the user clicks or tabs into the field.

5. This field is mandatory. If the user clicks 'Next' without entering anything in this field, display the following error message:

"Please enter a valid date, in the format DD/MM/YYYY"

6. This field requires specific data - when the user clicks 'Next', the input should be validated, using the following rules:

*Must be a valid date (i.e. 1-31 in the DD section, 1-12 in the MM section)

If an invalid date is entered, display the following error message:

"Please enter a valid date, in the format DD/MM/YYYY"

*Date entered cannot be in the future

If a date in the future is entered, display the error message "Please enter a date of birth in the past"

*Date entered cannot be less than 17 years prior to current date
*Date entered cannot be more than 101 years prior to current date

If date entered would make the user less than 17 or older than 101, display the message "Suncorp only provides home insurance for customers between 17 and 101 years of age"
